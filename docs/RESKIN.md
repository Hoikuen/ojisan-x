# RESKIN — イラストを差し替えれば別ゲームになる仕組み

作成日: 2026-06-26

このプロジェクトは「**リスキン可能なエンジン**」として作ってある。
ゲームのロジック（移動・ジャンプ・攻撃・つかみ・状態・当たり判定）は**キャラの見た目を一切知らない**。
画像（とフロアデータ・文言）を差し替えるだけで、**コード変更ゼロで別ゲームに化ける**。

3パターンくらい別ゲームを作る想定。共通エンジン＝これ、差分＝アート＋データ。

---

## 1. キャラを丸ごと別人にする（最小手順）

1. 新キャラを**同じポーズ一式**で描く（純白背景・足元をフレーム下端・右向き）。
   - 主人公：idle / walk(_1〜_4) / kick / crouch / crouch_attack / jump / fall / jump_attack / hurt / grabbed / death
   - 敵・ボスも同様（hug / card / umbrella …の各ポーズ）
2. `public/assets/sprites/raw_generated/` に置く（命名は `<キャラ>_<ポーズ>` 例: `ojisan_kick`）。
3. `python3 tools/extract_raw.py` を実行 → 透過抽出して `extracted_v2/...` の同じパスへ自動配置。
4. `npm run build` → そのまま新キャラが**同じ動きで**動く。

> 体型が人型なら当たり判定の数値（`TUNING` の displayHeight / bodyRatio）はそのままでOK。
> 極端に違う体型（横長・4足等）だけ `TUNING` を微調整。

---

## 2. 歩き等を「動いて」見せる（アニメーション）

各状態は **1コマ（静止画）でも複数コマ（アニメ）でも**よい。

- アニメさせたい動きは **連番**で複数枚生成：`<キャラ>_walk_1` `_walk_2` `_walk_3` `_walk_4`（2〜4枚でOK）。
- `extract_raw.py` が連番を検出し、**全コマを共通キャンバス（同サイズ・足元揃え）に整列**して出力（足元がブレない）。
- `src/data/assets.js` の `PLAYER_ANIMS` は **`normal`（通常版）と `bald`（パワーアップ版）の2セット**で、
  全ポーズ（idle/walk/punch/kick/crouch/crouch_attack/jump_attack/grabbed/hurt/death）が登録済み。
  各定義は `repeat:-1`=ループ（待機/歩行/もがき）、`repeat:0`=ワンショット（攻撃/被弾/やられ）。
  コマ数を増減したい時は `frames` 配列を足し引きするだけ（`BootScene` が起動時に自動でアニメ生成）。
- **コマが2枚未満の状態は自動で静止画（`PLAYER_TEX`）にフォールバック**（1枚でも必ず動く・壊れない）。
- 主人公スプライトの完全な発注リスト（通常28＋ハゲ化14＝42コマ）は `CHARACTER_DESIGN_PROMPTS.md` が正本。

---

## 3. 世界観・ステージも差し替え

- 背景：`background/` の画像を差し替え（タイトル＋各フロア）。
- ステージ構成：`src/data/floors.js`（ウェーブ・ボス・時間）を編集＝**データだけ**でフロア追加/変更。
- 画面文言：`docs/SCRIPT.md`（タイトル・物語・エンディング等）に沿ってテキスト差し替え。

---

## 4. 差し替えても変わらないもの（＝エンジン）

- 操作・物理・状態機械・つかみ・上下段回避・ビーム・スコア・残機・コンティニュー
- タッチ操作・HUD・カメラ・GitHub Pages自動デプロイ・Playwright自己テスト

→ **アート＋floors.js＋文言**を差し替えれば、同じ手触りの「別ゲーム」が出来上がる。
新ゲームを作るときは、このリポジトリを土台に上記だけ差し替える運用が最速。
